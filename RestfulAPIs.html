RESTful API Cheat Sheet

Request Methods

  • GET: Used to get data only and does not modify the data at all. Should return 200, 400, or 404 responses.
  • POST: Create a new resource (e.g., creating a new user). Should return 200, 201, or 204 responses.
  • DELETE: Delete a resource (e.g., delete a user). Should return 200, 202, or 204 responses.
  • PUT: Update a resource. If the resource doesnâ€™t exist, the API might decide to create it and return a 201 response. Should return 200, 201, or 204 responses.

Common Status Codes

2xx - Success Codes

  • 200 OK
  • 201 Created
  • 202 Accepted
  • 204 No Content

3xx - Redirection Codes

  • 301 Moved Permanently
  • 302 Found

4xx - Client Error Codes

  • 400 Bad Request
  • 401 Unauthorized
  • 403 Forbidden
  • 404 Not Found
  • 405 Method Not Allowed

5xx - Server Error Codes

  • 500 Internal Server Error
  • 501 Not Implemented

API keys are used as authentication credentials.
